01
The ID Process

Analyze the Problem

Before you open Claude, before you write a single prompt — stop. The #1 mistake people make with AI-generated training is skipping this step entirely. A beautiful module that solves the wrong problem is still a waste of everyone's time.

Step 1

Define the Performance Problem

Ask yourself: what is the actual behavior that needs to change? Not "what content do people need" but "what should someone do differently after this training?"

Claude Prompt
"I'm designing a training for [audience]. The performance problem is [specific gap]. Help me clarify: Is this a knowledge gap, a skill gap, a motivation gap, or a process/environment gap? Ask me questions to help diagnose the root cause."
ID Tip
If the root cause isn't a knowledge or skill gap, training isn't the solution. Claude can help you draft a recommendation memo to stakeholders explaining why a job aid, process fix, or tool change would be more effective.
Step 2

Define Your Learner Persona

Who is taking this training? What do they already know? What are their constraints (time, motivation, tech literacy)? The more specific you are, the better Claude's output will be.

Claude Prompt
"My learners are [role] at [company type]. They have [X level] of prior knowledge. Their biggest challenges are [pain points]. They have [X minutes] for training. Generate 2-3 detailed learner personas I can use to guide my design decisions."
Step 3

Write Performance-Based Learning Objectives

Not "learners will understand X" — that's unmeasurable. Use Bloom's Taxonomy action verbs tied to observable behaviors.

Claude Prompt
"Based on this performance problem: [gap], draft 3-5 learning objectives using Bloom's Taxonomy. Each should start with an action verb, describe an observable behavior, include conditions, and specify a success criterion. Make them measurable at Kirkpatrick Level 3 (on-the-job application)."
What You'll Get
Tight, measurable objectives like: "Given a customer complaint scenario, the learner will identify the root cause and select the appropriate resolution path with 90% accuracy" — not vague filler like "understand customer service principles."
02
The ID Process

Design the Experience

This is where most AI-generated training fails. People jump straight to "build me a course" without designing the learner journey first. Claude is powerful, but it needs your instructional design brain to guide it.

Step 4

Choose the Right Modality

Not everything should be an eLearning module. Use Action Mapping to decide: does this need a branching scenario? A job aid? A video? A blended approach? Match the solution to the problem.

Claude Prompt
"Here are my learning objectives: [paste them]. My learners are [persona]. They have [constraints]. Using Cathy Moore's Action Mapping framework, recommend the best modality for each objective. For each, explain why that format works better than alternatives."
Step 5

Create a Content Outline & Storyboard

Structure your module before you build it. Think: what's the narrative arc? Where are the decision points? Where does the learner practice?

Claude Prompt
"Create a detailed storyboard for a [X-minute] interactive training on [topic]. Include: screen-by-screen breakdown, content on each screen, interaction type (click-to-reveal, drag-and-drop, branching decision, knowledge check), navigation flow, and where branching scenarios split based on learner choices. Format as a table."
ID Tip
This is your blueprint. Review it carefully before moving to development. Every minute spent refining the storyboard saves 10 minutes of rework later. Share it with your SME for content validation.
Step 6

Script the Branching Scenario

If your training includes decision-based learning, script the full branching logic: decision points, consequences for each path, and feedback loops.

Claude Prompt
"Write a branching scenario for [topic]. The learner plays [role]. Scene: [realistic workplace situation]. At each decision point, offer 3 choices — one correct, one partially correct, one incorrect. Each choice leads to a realistic consequence and tailored feedback that explains why. Include 3 decision points with full branching paths."
03
Claude Design

Build It in Claude

Now comes the part Tim Slade demonstrated — turning your design into a functional, interactive learning experience using Claude's code generation. This is where the magic happens, but only because you did the thinking work first.

Step 7

Generate the Interactive Module

Use Claude to build a fully functional HTML/CSS/JS learning module based on your storyboard. Be specific about interactions, visual style, and structure.

Claude Prompt
"Build an interactive training module as a single-page HTML/CSS/JS application. Topic: [topic]. Structure: [paste your storyboard outline]. Include: progress bar, navigation (next/back), a branching scenario with [X] decision points, knowledge check questions with immediate feedback, a completion summary with score. Visual style: modern, clean, [brand colors]. Make it mobile responsive. Use smooth transitions between screens."
Pro Tip
Give Claude your full storyboard content in the same prompt. The more context you provide, the less you'll need to iterate. Include your exact text, scenario scripts, and quiz questions — don't make Claude guess.
Step 8

Add Rich Interactions

Claude can build interactions that rival traditional authoring tools: drag-and-drop, click-to-reveal, tabbed content, accordion panels, flip cards, timelines, and interactive infographics.

Claude Prompt
"Add these interactions to the module: 1) A drag-and-drop activity on screen 3 where learners match [X] to [Y]. 2) A flip-card review on screen 5 with 4 cards showing key concepts. 3) A branching scenario on screens 7-10 based on this script: [paste branching scenario]. Include animated transitions and immediate feedback for each interaction."
Step 9

Build the Assessment

Create knowledge checks that test application, not recall. Claude can build scenario-based assessments with scoring, feedback, and retry logic.

Claude Prompt
"Create a 5-question scenario-based assessment for this module. Each question should present a realistic workplace situation where the learner applies what they learned. Include: the scenario context, 4 answer options, correct answer with explanation, incorrect answer feedback that guides without giving away the answer, a running score counter, and a final results screen with pass/fail (80% threshold) and a personalized summary."
Step 10

Polish the Visual Design

This is where Claude Design shines — creating production-quality visuals that look like they came from a design team, not a template.

Claude Prompt
"Redesign the visual style of this module. Apply: [brand colors/hex codes], consistent typography (use Google Fonts), custom icons for navigation, a professional header with module title and progress indicator, smooth CSS animations between screens, and a cohesive color system. It should look like a professional LMS course, not a basic HTML page."
What You'll Get
A polished, interactive training module that rivals Articulate Rise output — with branching logic, assessments, animations, and responsive design. All in a single HTML file you can host anywhere.
04
Quality Assurance

Iterate & Refine

Your first output won't be perfect. The real power of Claude is how fast you can iterate. This is where the ID process and the tool converge.

Step 11

QA Check: Content Accuracy

Review every screen against your storyboard. Check facts with your SME. Verify that branching paths lead to the right consequences.

Claude Prompt
"Review this module for: 1) Content accuracy — flag anything that seems inconsistent with [topic area]. 2) Instructional soundness — are the interactions aligned with the learning objectives? 3) Flow — does the narrative make sense? 4) Accessibility — are there alt-text descriptions, keyboard navigation, sufficient color contrast?"
Step 12

QA Check: Accessibility (WCAG)

Ensure your training is inclusive — screen reader compatible, keyboard navigable, high contrast, and readable at different zoom levels.

Claude Prompt
"Audit this module for WCAG 2.1 AA compliance. Add: aria-labels to all interactive elements, keyboard navigation support (tab, enter, escape), focus indicators, alt text for any images, sufficient color contrast ratios (4.5:1 minimum for text), and a skip navigation link."
Step 13

Pilot Test with Real Learners

Before full deployment, run your module with 3-5 representative learners. Watch them use it. Note where they get confused, stuck, or disengaged.

ID Tip
This step is non-negotiable. No amount of AI-generated polish replaces watching a real human interact with your training. Their confusion IS your redesign brief. Feed what you learn back into Claude for rapid fixes.
05
Distribution

Deploy & Distribute

You've built a standalone HTML learning experience. Here's how to get it in front of learners.

Step 14

Hosting Options

Your Claude-built module is a single HTML file. That gives you flexibility traditional authoring tools don't.

Options
Company intranet/SharePoint: Upload the HTML file directly. Learners access via link.

LMS with SCORM: Ask Claude to wrap the module in a SCORM 1.2 or 2004 package with completion tracking. This is solvable — Tim Slade mentioned it too.

GitHub Pages: Free hosting for public-facing training or portfolio pieces.

Articulate Reach 360 / LMS: If your org uses these, export accordingly.
Claude Prompt (SCORM Wrapper)
"Wrap this HTML module in a SCORM 1.2 compliant package. Include: imsmanifest.xml, SCO tracking for completion status (complete/incomplete), score reporting from the assessment, and a zip file structure I can upload directly to an LMS."
06
The Part Most People Skip

Measure Impact

This is where 90% of training falls apart. You built something beautiful. Did it work? Claude can help you design the measurement strategy — but you need to build it in from the start, not bolt it on after.

Step 15

Design Your Evaluation Framework

Use Kirkpatrick's model, but don't stop at Level 1 (did they like it) or Level 2 (did they pass the quiz).

Claude Prompt
"Design a Kirkpatrick L1-L4 evaluation plan for this training. Include: L1 — a 5-question reaction survey with specific, actionable questions (not just 'did you enjoy it'). L2 — pre/post knowledge assessment design. L3 — a 30-day follow-up manager survey to assess on-the-job behavior change. L4 — business metrics to track (tie to the original performance problem). Give me specific survey questions for each level."
Step 16

Build the Surveys in Claude

Claude can generate interactive survey forms you can embed or distribute separately.

Claude Prompt
"Build an interactive HTML survey for the L1 reaction evaluation. Include 5 questions: 3 Likert scale, 1 open-text, 1 NPS-style. Make it visually consistent with the training module. Include a submit button that shows a thank-you message. If possible, add the ability to export responses."
The Real Question
When you present results to stakeholders, they don't want to know completion rates. They want to know: did this training solve the problem we identified in Step 1? That's why the analysis phase matters so much — it defines what "success" means before you build anything.
